根据用户指定的关键字段匹配不同数据源

重量信息：实际重量、预估重量、计费重量（取大值）

费用对比：理论运费、账单金额、差异金额

扣款项：撕单标识、赔付金额

应付金额：理论运费 - 扣款

备注：匹配失败原因、报价规则说明等

支持数据透视汇总（按云仓、快递公司汇总差异）

匹配计算模块：毛重匹配、报价匹配、费用计算、账单比对、扣款关联

输出模块：生成结果工作表，汇总统计

针对您提出的两个关键难点，我们进一步细化技术方案：

难点1：旺店通明细与云仓账单的字段列位置不固定
解决方案：灵活的字段映射机制
在字段映射界面，用户可以从源表的所有列标题中选择对应系统字段。

系统字段列表预定义了所有必需的字段（如物流单号、货品编码、数量、收件省份、发货时间、实际重量等），以及可选的扩展字段。

映射存储：将系统字段名与源表中的列索引（或列名）的对应关系保存在字典中。后续读取数据时，直接根据索引提取，无需硬编码列号。

容错处理：若某行缺少关键字段（如物流单号为空），则跳过并记录错误。

难点2：重量按报价区间向上取整后进行多条件匹配
详细算法流程
步骤1：获取订单的“计费重量”
优先使用实际重量：若旺店通明细中提供了实际重量（且用户勾选“使用实际重量”），则取实际重量。

否则通过毛重表估算：根据订单中的货品编码和数量，在毛重表中查找单个毛重，累加得到预估总重。

体积重量判断：若订单有体积信息（长宽高），根据报价明细中的“计泡体积”规则（如长×宽×高÷6000）计算体积重量，并与实际/预估重量比较，取较大值作为计费重量。

步骤2：按报价单的区间向上取整
报价单的“重量区间段结束”列：通常表示该区间适用的最大重量（如0.5、1、2、3…）。实际计费时，需要将订单重量“向上舍入”到最近的区间结束值。

例如：重量1.2kg，报价区间结束为0.5、1、2、3…，则向上取整到2kg（因为1.2 > 1，所以适用2kg区间）。

实现方法：

预先提取报价明细中所有唯一的重量区间结束值，并排序（升序）。
对于每个订单的计费重量 w，在排序后的数组中找到第一个 区间结束值 >= w 的值，即为取整后的计费重量。
若 w 大于最大区间结束值，则可能需要特殊处理（如按超重计费或提示）。
步骤3：多条件匹配报价规则
匹配条件：云仓、快递公司、地区、是否打包品、发货时间（必须在生效/失效范围内）、取整后的计费重量。

匹配逻辑：

根据云仓、快递公司、发货时间筛选出候选报价行。
地区匹配：支持精确匹配、通配符（如“*”表示全国），或省/市模糊匹配。建议将报价单中的地区统一为规范名称（如去除“省”、“市”），并与订单中的收件省份进行比对。
重量匹配：取整后的计费重量必须落在该报价行的区间内（即重量 ≤ 区间结束值）。注意：报价单中可能有多条相同重量区间但不同生效日期的记录，需按生效时间取最新的一条。
是否打包品匹配：若订单包含打包品（可通过货品属性判断），则匹配“是”，否则匹配“否”或“不限”（报价单中可能设置“是否打包品”为“*”表示都适用）。
步骤4：计算理论运费
根据报价单的计费结构计算：

阶梯单价：直接按取整后的重量 × 单价。

首重+续重：如首重1kg内价格A，续重每kg价格B。则运费 = A + max(0, (取整重量 - 首重)) × B。

若报价单中同时包含包装费等其他费用，则累加。

特殊情况处理
无匹配报价：标记为“待人工处理”，并在结果中注明可能原因（如地区无报价、重量超限等）。

多条报价匹配：默认取生效时间最晚的，或按用户预设的优先级（如优先匹配精确地区，再匹配全国）。

Dim weightBreaks() As Double
weightBreaks = GetSortedWeightBreaks(arrBaoJia)

Function GetMatchedPrice(wl As String, kuaidi As String, province As String, weight As Double, shipDate As Date, isPack As Boolean) As Double
    ' 1. 向上取整重量
    Dim w_ceil As Double
    w_ceil = CeilToBreak(weight, weightBreaks)
    
    ' 2. 构建查询键
    Dim key As String
    key = wl & "|" & kuaidi & "|" & province & "|" & IIf(isPack, "是", "否")
    
    ' 3. 查找匹配的报价行列表
    Dim rows As Variant
    If dictBaoJia.Exists(key) Then
        rows = dictBaoJia(key)  ' 假设存储为二维数组，每行一条报价规则
    Else
        ' 尝试用通配符地区（如全国）
        key = wl & "|" & kuaidi & "|*|" & IIf(isPack, "是", "否")
        If dictBaoJia.Exists(key) Then rows = dictBaoJia(key) Else rows = Null
    End If
    
    If IsNull(rows) Then
        GetMatchedPrice = -1  ' 表示无匹配
        Exit Function
    End If
    
    ' 4. 在rows中找到生效时间符合shipDate且重量区间包含w_ceil的最新规则
    Dim i As Long, matchedRow As Variant
    matchedRow = Null
    For i = 1 To UBound(rows, 1)
        If shipDate >= rows(i, 生效时间列) And shipDate <= rows(i, 失效时间列) Then
            If w_ceil <= rows(i, 重量区间结束列) Then
                ' 取第一条（假设已按生效时间降序排序）
                matchedRow = rows(i, 1)
                Exit For
            End If
        End If
    Next i
    
    If IsNull(matchedRow) Then
        GetMatchedPrice = -1
    Else
        ' 5. 计算运费（根据matchedRow中的价格字段）
        GetMatchedPrice = matchedRow(首重价格列) + (w_ceil - matchedRow(首重重量列)) * matchedRow(续重价格列)
    End If
End Function

Function CalcFee(weight As Double, row As Variant) As Double
    Dim firstWeight As Double, firstPrice As Double, unitPrice As Double
    firstWeight = row(首重重量列)
    firstPrice = row(首重价格列)
    unitPrice = row(续重单价列)
    
    If weight <= row(重量区间结束列) Then
        ' 正常区间内
        If weight <= firstWeight Then
            CalcFee = firstPrice
        Else
            CalcFee = firstPrice + (weight - firstWeight) * unitPrice
        End If
    Else
        ' 超过区间结束，按延伸处理（需确保已配置）
        ' 注意：这里传入的row已经是最大区间行
        CalcFee = firstPrice + (weight - firstWeight) * unitPrice
    End If
End Function

Function IsPackedOrder(orderRows As Variant, dictGuiGe As Dictionary) As Boolean
    ' orderRows: 二维数组，包含该订单的所有货品行
    ' 返回 True 表示打包品（存在散装），False 表示非打包品（全整箱）
    Dim i As Long, code As String, qty As Double, guige As Double
    For i = LBound(orderRows, 1) To UBound(orderRows, 1)
        code = orderRows(i, 货品编码列索引)
        qty = orderRows(i, 数量列索引)
        If dictGuiGe.Exists(code) Then
            guige = dictGuiGe(code)
            If guige > 0 Then
                If qty / guige - Int(qty / guige) <> 0 Then
                    ' 存在非整箱
                    IsPackedOrder = True
                    Exit Function
                End If
            Else
                ' 箱规为0或未定义，按非整箱处理（可配置）
                IsPackedOrder = True
                Exit Function
            End If
        Else
            ' 无箱规信息，按非整箱处理（保守）
            IsPackedOrder = True
            Exit Function
        End If
    Next i
    IsPackedOrder = False
End Function

Function MapNameToCode(rowsData As Variant, dictNameToCode As Dictionary, nameColIndex As Integer, codeColIndex As Integer) As Variant
    ' rowsData 按引用传递，在原数组增加一列存放货品编码
    ' 返回新增列的索引
    Dim i As Long, name As String
    For i = LBound(rowsData, 1) To UBound(rowsData, 1)
        name = Trim(rowsData(i, nameColIndex))
        If dictNameToCode.Exists(name) Then
            rowsData(i, codeColIndex) = dictNameToCode(name)
        Else
            rowsData(i, codeColIndex) = "未匹配"
        End If
    Next i
End Function

导入SQLite的必要性
性能：SQLite对结构化数据有索引优化，关联查询速度远快于VBA在内存中循环。

内存占用：数据存储在数据库文件中，不占用Excel内存，可处理百万行级别。

灵活性：可使用复杂的SQL进行多表连接、分组、条件计算，简化VBA代码。

导入方式建议
由于数据量可能达到100万行，逐行插入会非常慢，需采用批量插入策略。

方案一：使用事务+多行INSERT（推荐）
将一批数据拼接成一条INSERT语句插入多行，每批几千行，并使用事务包裹。

vba
' 示例：批量插入旺店通数据到SQLite表
Sub BulkInsertToSQLite(dataArr As Variant, tableName As String, conn As Object)
    ' dataArr: 二维数组，包含数据（假设第一行是标题，实际数据从第二行开始）
    ' 假设表结构已创建好
    Dim i As Long, j As Long
    Dim sql As String, values As String
    Dim batchSize As Long: batchSize = 1000
    Dim rowCount As Long: rowCount = UBound(dataArr, 1) - LBound(dataArr, 1) + 1
    
    ' 开启事务
    conn.Execute "BEGIN TRANSACTION"
    
    For i = LBound(dataArr, 1) + 1 To UBound(dataArr, 1)  ' 跳过标题行
        ' 构建一行值的字符串：('值1','值2',...)
        values = "("
        For j = LBound(dataArr, 2) To UBound(dataArr, 2)
            ' 处理空值和转义单引号
            Dim val As String
            If IsNull(dataArr(i, j)) Or dataArr(i, j) = "" Then
                val = "NULL"
            Else
                val = "'" & Replace(dataArr(i, j), "'", "''") & "'"
            End If
            values = values & val & IIf(j = UBound(dataArr, 2), "", ",")
        Next j
        values = values & ")"
        
        ' 拼接SQL，每1000行执行一次
        If i Mod batchSize = 0 Or i = UBound(dataArr, 1) Then
            ' 执行当前累积的SQL
            sql = "INSERT INTO " & tableName & " VALUES " & values & ";"
            conn.Execute sql
            sql = ""  ' 重置
        Else
            ' 累积values，用逗号连接
            If sql = "" Then
                sql = "INSERT INTO " & tableName & " VALUES " & values
            Else
                sql = sql & "," & values
            End If
        End If
    Next i
    
    ' 提交事务
    conn.Execute "COMMIT"
End Sub

方案5：通过WebService中转（最灵活）

function querySQLite(sql) {
    let xhr = new XMLHttpRequest();
    xhr.open("POST", "http://localhost:8080/query", false);
    xhr.send(JSON.stringify({sql: sql}));
    return JSON.parse(xhr.responseText);
}
优点：跨平台、可多人共享、可实现复杂逻辑
缺点：需要搭建和维护Web服务

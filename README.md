# 多云仓自动对账工具（MVP）

基于 Streamlit 的主对账功能 MVP，固定字段名，快速验证业务流程。

## 运行
```powershell
python -m pip install -r requirements.txt
python -m streamlit run app.py
```

## Sheet 选择与性能（重要）
- 工具内所有涉及 Excel 上传的地方，均支持“同一工作簿多选 Sheet”。选中的多个 Sheet 会被合并处理（concat）。
- 性能策略：上传后会先解析 Sheet 名与表头；在点击执行/生成时才读取明细数据（大表场景会显示读取进度）。

## 多文件汇总（已支持）

### 发货明细
- 支持**多文件上传**后自动汇总。
- 支持**旺店通发货明细 + 云宝发货明细同时上传**，会先合并再对账。
- 若上传了云宝发货明细，仍需要上传“云宝名称货品表”用于把“货品名称”映射到“商家编码”。

### 云仓账单
- 支持**多文件上传**后自动汇总（例如同一云仓按快递拆分多个账单）。
- 同一“物流单号 + 云仓”在多个账单文件中出现时，会先汇总：快递费/包装费求和，计费重量取最大。

## 账单存在但明细缺失（已支持）
- 即使不上传发货明细，只要上传了云仓账单，也可以生成结果。
- 若某些物流单号只在账单中存在、在发货明细中不存在：
	- 仍会出现在“对账结果”中
	- “备注”会标记为“发货明细缺失”
	- 同时会出现在“异常清单”中

## 固定字段名（必需）

### 旺店通发货明细
- 物流单号
- 商家编码
- 数量
- 收货省份
- 发货时间
- 快递公司
- 实际重量（可选）
- 店铺名称（可选）

### 云宝发货明细
- 物流单号
- 货品名称
- 数量
- 收货地址
- 发货时间
- 快递公司
- 实际重量（可选）
- 店铺名称（可选）

### 云宝名称货品表
- 货品名称
- 商家编码

### 毛重表
- 商家编码
- 毛重(g)
- 箱规

### 云仓重量段定义表
- 云仓
- 重量段结束(kg)

### 多条件资费表
- 云仓
- 快递公司
- 重量段结束(kg)
- 是否打包品
- 省份
- 生效开始日期
- 生效结束日期
- 首重(kg)
- 首费(元)
- 续重(kg)
- 续费(元)

> 说明：当“重量段结束(kg)=0”时，表示【超重计费规则行】（用于超过最大重量段时按“首重/续重”递增计费）。

### 云仓账单
- 物流单号
- 计费重量(kg)
- 快递费(元)
- 包装费(元)（可选）
- 云仓

### 可选表
- 耗材表：商家编码、价格(元)
- 撕单表：物流单号
- 售后赔付表：物流单号、赔付金额(元)

## 输出
- 对账结果：明细对账与核算运费
- 汇总：按云仓+快递公司汇总
- 异常清单：资费未匹配、重量段缺失、账单缺失

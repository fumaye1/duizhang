# 多云仓自动对账工具（MVP）

基于 Streamlit 的主对账功能 MVP，固定字段名，快速验证业务流程。

## 运行
```powershell
python -m pip install -r requirements.txt
python -m streamlit run app.py
```

## Sheet 选择与性能（重要）
- 工具内所有涉及 Excel 上传的地方，均支持“同一工作簿多选 Sheet”。选中的多个 Sheet 会被合并处理（concat）。
- 性能策略：上传后会先解析 Sheet 名与表头；在点击执行/生成时才读取明细数据（大表场景会显示读取进度）。

## 多文件汇总（已支持）

### 发货明细
- 支持**多文件上传**后自动汇总。
- 支持**旺店通发货明细 + 云宝发货明细同时上传**，会先合并再对账。
- 若上传了云宝发货明细，仍需要上传“云宝名称货品表”用于把“货品名称”映射到“商家编码”。

### 云仓账单
- 支持**多文件上传**后自动汇总（例如同一云仓按快递拆分多个账单）。
- 同一“物流单号 + 云仓”在多个账单文件中出现时，会先汇总：快递费/包装费求和，计费重量取最大。

## 账单存在但明细缺失（已支持）
- 即使不上传发货明细，只要上传了云仓账单，也可以生成结果。
- 若某些物流单号只在账单中存在、在发货明细中不存在：
	- 仍会出现在“对账结果”中
	- “备注”会标记为“发货明细缺失”
	- 同时会出现在“异常清单”中

## 规范字段（工具内部使用）

源表列名不要求完全一致，但需要在“字段映射（拖拽模态框）”中把源字段映射到下列规范字段。
若源表列名刚好与规范字段一致，也可以直接勾选并排序后完成配对。

### 旺店通发货明细
- 物流单号
- 商家编码
- 数量
- 收货省份
- 发货时间
- 快递公司
- 预估重量(kg)（可选：当对账参数选择“使用发货明细预估重量”时需要）
- 实际重量（可选）
- 店铺名称（可选）

### 云宝发货明细
- 物流单号
- 货品名称
- 数量
- 收货地址
- 发货时间
- 快递公司
- 预估重量(kg)（可选：当对账参数选择“使用发货明细预估重量”时需要）
- 实际重量（可选）
- 店铺名称（可选）

### 云宝名称货品表
- 货品名称
- 商家编码

### 毛重表
- 商家编码
- 毛重(g)
- 箱规

### 云仓重量段定义表
- 云仓
- 重量段结束(kg)

### 多条件资费表
- 云仓
- 快递公司
- 重量段结束(kg)
- 是否打包品
- 省份
- 生效开始日期
- 生效结束日期
- 首重(kg)
- 首费(元)
- 续重(kg)
- 续费(元)

> 说明：当“重量段结束(kg)=0”时，表示【超重计费规则行】（用于超过最大重量段时按“首重/续重”递增计费）。

### 云仓账单
- 物流单号
- 计费重量(kg)
- 快递费(元)
- 包装费(元)（可选）
- 云仓

### 可选表
- 耗材表：商家编码、价格(元)
- 撕单表：物流单号
- 售后赔付表：物流单号、赔付金额(元)

## 输出
- 对账结果：明细对账与核算运费
- 汇总：按云仓+快递公司汇总
- 异常清单：资费未匹配、重量段缺失、账单缺失

## 对账参数说明

### 重量来源
- 使用毛重表核算重量（kg）：按 `数量×毛重(g)/1000` 计算“计费重量原始”，再按“重量段定义表”取整核算。
- 使用发货明细预估重量（kg）：在页面参数区从发货明细表头中选择一列作为 `预估重量(kg)`，再按“重量段定义表”取整核算。

### 是否打包品匹配
- 可选择“按是否打包品匹配资费”（需要毛重表提供 `箱规`）。
- 若不启用，可指定默认口径：打包品 / 非打包品 / 不限（忽略是否打包品条件）。

## 字段映射（拖拽模态框）

各数据源映射区提供“打开拖拽映射（模态对话框）”按钮：

- 左侧为“源字段”，右侧为“规范字段”。
- 每侧分为“已勾选 / 未勾选”两栏容器：
	- 将字段拖到“已勾选”即视为勾选；拖回“未勾选”即取消。
	- “已勾选”会自动前置显示。
- 最终映射按两侧“已勾选”的顺序一一配对（按最短长度配对）。
- 规范字段侧默认勾选“必填规范字段”，且必填集合可在模态框顶部自定义。
- 为避免鼠标悬浮引起闪烁，字段项无 hover 宽度特效（悬浮不改变尺寸）。

> 说明：若运行环境缺少拖拽组件依赖，会自动降级为“勾选 + 顺序配对”的原生方案。

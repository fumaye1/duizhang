## 本轮对话主要完成的功能

### 1) 主对账 MVP（Streamlit）
- 新增 `app.py`，完成主流程单页 UI（对照 HTML Demo 的主对账结构）。
- 支持 ERP 类型切换：`旺店通` / `云宝`。
- 支持上传并读取 Excel（`.xlsx/.xls`），按 Sheet 选择读取。
- 支持主流程参数配置：云仓、重量来源、是否按打包品匹配资费、是否启用扣减、是否启用耗材。

### 2) 对账核心计算逻辑（固定字段名）
- 新增 `reconcile.py`，实现：
  - 数据字段校验（`REQUIRED_COLUMNS` + `validate_columns`）。
  - 云宝货品名称映射为商家编码。
  - 省份清洗标准化。
  - 重量来源可配置：使用发货明细预估重量（kg）或使用毛重表核算重量（kg）（暂不含计泡体积）。
  - 结算重量按“重量段结束(kg)”向上匹配。
  - 资费匹配（云仓/快递公司/重量段/省份/生效期），可选是否把“是否打包品”作为匹配条件。
  - 资费计算规则：
    - 支持新资费结构 `首重(kg)/首费(元)/续重(kg)/续费(元)`，并支持 `重量段结束(kg)=0` 作为“超重最大区间”规则。
    - 非0段按“最小覆盖段（重量段结束>=结算重量）”匹配；超重时走0段按“首重+续费”计算。
    - 兼容旧资费字段（`快递费(元)`/`快递费`/`首重价格+续重价格`）作为兜底。
  - 撕单扣减、售后赔付扣减。
  - 云仓账单关联、差异金额与应付金额计算。
  - 输出三张结果：`对账结果` / `汇总` / `异常清单`。

### 3) 字段映射 UI
- 已在 `app.py` 中为主要数据源增加字段映射区：
  - 发货明细、云宝名称货品表、毛重表、重量段定义表、资费表、账单、耗材/撕单/售后。
- 支持：
  - 必填字段映射校验（未映射阻止执行）。
  - 映射列重复检测（同一源列被重复映射会提示并阻止执行）。

### 4) 字段映射模板保存/加载

- 支持 JSON 模板上传加载（自动回填到映射下拉框）。
- 支持当前映射导出为 JSON 模板下载。
- `README.md` 已补充模板使用说明与示例 JSON。

### 5) 文档与可运行性补充

### 6) 多文件汇总导入（发货明细/云仓账单）
- 发货明细支持**多文件上传**并汇总：
  - 旺店通发货明细：多文件合并后再对账。
  - 云宝发货明细：多文件合并后再对账，并支持与旺店通明细**同一次对账合并**。
- 云仓账单支持**多文件上传**并汇总：
  - 适配“同一云仓按快递拆分多个账单文件”的场景。
  - 账单汇总口径：同一 `物流单号 + 云仓`，快递费/包装费求和，计费重量取最大。

### 7) 账单存在但发货明细缺失也输出
- 若发货明细不存在、但云仓账单存在：允许执行并在结果中输出账单行。
- 若某些单号仅存在于账单：仍会出现在“对账结果”里，并在“异常清单”标记为“发货明细缺失”。

### 8) 省份清洗逻辑增强（直辖市）
- 对“北京/上海/天津/重庆”直辖市，清洗后统一提取为不带“市”的名称：
  - 例如：`北京市 北京市 昌平区` -> `北京`；`北京 北京市 昌平区` -> `北京`。

### 9) 多仓快递费汇总与回冲（首版）
- 在 `app.py` 新增 Tab：`📦 多仓汇总与回冲`。
- 多仓快递费汇总：支持上传多个“对账结果表”，按默认字段预选 + 可手工选择字段，导出汇总 Excel。
- 多仓汇总分页（分Sheet）：当汇总行数超过阈值（默认 80 万，可配置）时，按 `云仓` 拆分为多个 Sheet；若单仓仍超阈值则继续按阈值切分。
- 回冲差异分析：支持上传“新复核结果表（可多选）”+“旧汇总表”，按物流单号汇总后对比选定字段，导出差异表（回冲表）。
- 为支持多仓场景，补齐对账结果导出关键字段并强化账单关联：
  - `reconcile.py` 正常对账结果导出列增加 `云仓/快递公司/账单计费重量/账单包装费`。
  - 云仓账单与明细关联由仅 `物流单号` 调整为 `物流单号 + 云仓`，避免跨云仓串账。

### 9.1) 多仓汇总与回冲（2026-03-06 迭代：更稳 + 更快）
- 字段口径调整：
  - **不做任何自动字段清洗/同义词归一**；上传前要求用户在源表中手动规范化列名。
  - 上传后只做“字段命中检查”（同名即命中），提示每个文件/Sheet 的未命中字段。
  - **未全部命中时禁止生成**（汇总/回冲均如此），避免工具内部映射导致口径漂移。
- 上传能力增强：
  - 对账结果文件、新复核结果表：支持“同一工作簿多选 Sheet”，每个选中 Sheet 都参与后续生成。
- 性能优化（大表场景）：
  - 分阶段懒解析：先读 Sheet 名 → 再仅读表头（`nrows=0`）检查列名 → 点击生成时按 `usecols` 只读所需字段。
  - 增加进度显示：表头检查进度、生成时读取明细进度。

### 9.2) 上传解析逻辑全模块统一（2026-03-06 迭代：一致 + 可感知进度）
- 目标：让“上传文件→选Sheet→读取数据”的体验与性能策略在各模块一致（不只多仓模块）。
- 覆盖范围：
  - 主对账：云仓账单/发货明细/毛重表/重量段定义表/资费表/可选表等上传均支持“同一工作簿多选 Sheet”。
  - 回冲差异：旧汇总表也支持多选 Sheet。
  - 快递费分析：对账结果表支持多选 Sheet。
  - 汇总账单：结果表支持多选 Sheet。
- 性能与反馈：
  - 统一采用分阶段懒解析（Sheet名→表头→按需读取）。
  - 在点击“开始对账/生成分析/生成账单”等入口读取数据时增加读取进度与当前文件/Sheet 状态提示。

### 10) 快递费分析模块（首版，ModuleAnalysis）
- 在 `app.py` 新增 Tab：`📊 快递费分析`，基于“单个云仓对账结果表”生成分析报表。
- 新增 `analysis_module.py`（纯函数模块，便于复用/迁移）：
  - 运营交割价配置解析（重量上限→价格）。
  - 省份透视：单量透视 + 占比透视。
  - TOP 单品：按商家编码汇总单量/费用/均价，并结合交割价计算差异。
  - 重量单价表：按结算重量汇总单量/费用/均价，并结合交割价计算差异。
- 支持将上述分析结果导出为多 Sheet Excel。

### 11) 汇总账单模块（首版，ModuleBillSummary-简化版）
- 在 `app.py` 新增 Tab：`🧾 汇总账单`，根据“结果表”生成账单 DataFrame：`序号/项目/金额/余额`。
- 期初余额：支持手动输入；或通过“WPS多维表API（通用HTTP取数配置）”在线获取。
- 系数口径：结果表项目行默认系数为 `1`；本月预充固定按 `-1` 系数计入。
- 项目行：支持从结果表中手动选择若干字段并求和（如快递费、耗材费、加收费等），写入“金额”，并按 `余额 = 上一行余额 + 本行金额` 滚动计算。
- 本月预充：支持手动输入或通过“WPS多维表API（通用HTTP取数配置）”在线获取；作为最后一个项目行（生成账单时按负数计入）。
- 总计行：`余额` 等于预充行余额。
- 新增模块文件：
  - `bill_summary_module.py`：汇总账单生成纯函数。
  - `wps_http_client.py`：可配置HTTP取数（用于对接WPS多维表API返回的数值）。

### 12) 主对账重量/打包品口径增强（2026-03-07）
- 重量来源升级为“二选一”：
  - 使用毛重表核算重量（kg）：按 `数量×毛重(g)/1000` 作为计费重量原始，再按重量段取整。
  - 使用发货明细预估重量（kg）：从发货明细表头中手动选择一列作为预估重量(kg)，系统自动注入字段映射并用于取整核算。
- 打包品匹配升级为“可选条件”：
  - 可启用“按是否打包品匹配资费”（需要毛重表箱规）。
  - 若不启用，可设置默认口径：打包品 / 非打包品 / 不限（忽略是否打包品条件）。
- 毛重表不再一律必传：仅在“重量核算选择毛重表”或“启用打包品判断”时才必传。

## 调试过程记录（日期：2026-03-04）

### 1) 页面无法访问
- 现象：浏览器提示 `localhost 拒绝连接`。
- 排查：检查 Streamlit 启动终端输出，发现卡在 Streamlit 首次启动的 `Email:` 输入提示。
- 原因：进程等待交互输入，服务尚未真正监听端口。
- 处理：
  - 方案1：在该终端直接回车跳过邮箱输入。
  - 方案2：使用参数禁用统计提示重新启动：
    - `python -m streamlit run app.py --browser.gatherUsageStats false`
- 结果：服务可启动，页面可通过 `http://localhost:8501` 访问。

### 2) 功能增量调试
- 字段映射 UI 引入后，进行了语法检查并确认无语法错误。
- 模板加载容错：对 JSON 解析失败增加错误提示（格式错误/解析失败）。

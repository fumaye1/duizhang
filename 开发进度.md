# 多云仓自动对账工具 - 开发进度文档

## 日期
- 2026-03-04

## 本轮对话主要完成的功能

### 1) 主对账 MVP（Streamlit）
- 新增 `app.py`，完成主流程单页 UI（对照 HTML Demo 的主对账结构）。
- 支持 ERP 类型切换：`旺店通` / `云宝`。
- 支持上传并读取 Excel（`.xlsx/.xls`），按 Sheet 选择读取。
- 支持主流程参数配置：云仓、是否优先实际重量、是否启用扣减、是否启用耗材。

### 2) 对账核心计算逻辑（固定字段名）
- 新增 `reconcile.py`，实现：
  - 数据字段校验（`REQUIRED_COLUMNS` + `validate_columns`）。
  - 云宝货品名称映射为商家编码。
  - 省份清洗标准化。
  - 预估重量/实际重量取值逻辑（暂不含计泡体积）。
  - 结算重量按“重量段结束(kg)”向上匹配。
  - 资费匹配（云仓/快递公司/重量段/是否打包品/省份/生效期）。
  - 资费价格优先级：`快递费(元)` -> `快递费` -> `首重价格+续重价格`。
  - 撕单扣减、售后赔付扣减。
  - 云仓账单关联、差异金额与应付金额计算。
  - 输出三张结果：`对账结果` / `汇总` / `异常清单`。

### 3) 字段映射 UI
- 已在 `app.py` 中为主要数据源增加字段映射区：
  - 发货明细、云宝名称货品表、毛重表、重量段定义表、资费表、账单、耗材/撕单/售后。
- 支持：
  - 必填字段映射校验（未映射阻止执行）。
  - 映射列重复检测（同一源列被重复映射会提示并阻止执行）。

### 4) 字段映射模板保存/加载
- 在 UI 顶部新增“字段映射模板”区域。
- 支持 JSON 模板上传加载（自动回填到映射下拉框）。
- 支持当前映射导出为 JSON 模板下载。
- `README.md` 已补充模板使用说明与示例 JSON。

### 5) 文档与可运行性补充
- `README.md` 已更新运行命令、固定字段要求、输出说明、模板格式。
- 执行过语法校验：
  - `python -m py_compile app.py`
  - `python -m py_compile reconcile.py`

## 调试过程记录（日期：2026-03-04）

### 1) 页面无法访问
- 现象：浏览器提示 `localhost 拒绝连接`。
- 排查：检查 Streamlit 启动终端输出，发现卡在 Streamlit 首次启动的 `Email:` 输入提示。
- 原因：进程等待交互输入，服务尚未真正监听端口。
- 处理：
  - 方案1：在该终端直接回车跳过邮箱输入。
  - 方案2：使用参数禁用统计提示重新启动：
    - `python -m streamlit run app.py --browser.gatherUsageStats false`
- 结果：服务可启动，页面可通过 `http://localhost:8501` 访问。

### 2) 功能增量调试
- 字段映射 UI 引入后，进行了语法检查并确认无语法错误。
- 模板加载容错：对 JSON 解析失败增加错误提示（格式错误/解析失败）。
